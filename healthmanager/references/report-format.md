# HealthManager 报告格式

版本：2026-03-09

## 输出模式

### 1. summary

适用场景：
- 每日定时巡检
- 早间提醒
- 用户只想快速知道有没有问题

默认结构：
1. 今日结论
2. 风险等级汇总
3. 关键风险
4. 当前系统状态
5. 今日建议动作

写法要求：
- 先说结论
- 尽量控制在短文本
- 只展开最重要的 1 到 3 个点
- 不要堆所有检查细节

### 2. full

适用场景：
- 首次审计
- 手动巡检
- 用户明确要求详细报告
- 需要留档

默认结构：
1. 总览
2. 风险等级汇总
3. OpenClaw 安全审计
4. 版本状态
5. 防火墙
6. 监听端口
7. 文件权限
8. 登录记录
9. 需关注项
10. 今日优先处理建议

## 写法要求

### 总览或今日结论
先用 1 到 3 句讲清：
- 当前整体是否正常
- 有没有需要优先处理的问题
- 这份报告的主结论是什么

### 风险等级汇总
建议用这种压缩表达：
- 1 Critical · 4 Warn · 1 Info

### 关键风险
优先写最核心的安全项：
- 插件风险
- 网关暴露
- 权限扩张
- 配置不当
- 可信边界问题

### 版本状态
说明：
- 当前版本
- 更新渠道
- 是否落后
- 是否有异常

### 防火墙
说明：
- 是否启用
- 是否有明显风险

### 监听端口
说明：
- 哪些端口在监听
- 是否仅本地
- 是否有公网暴露

### 文件权限
优先看：
- OpenClaw 配置
- 凭据文件
- 关键 JSON
- 敏感目录权限

### 登录记录
说明：
- 最近是否有异常远程登录
- 是否只是本地 TTY

### 需关注项
用列表给出：
- 立即处理
- 建议处理
- 可观察

### 今日优先处理建议
最多给 1 到 3 条。
如果只有一个最重要的问题，就只给一个。

## 语言要求

- 先结论后细节
- 直接，不绕
- 不要空话
- 不要只报问题，要给动作
